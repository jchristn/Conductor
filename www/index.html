The core abstraction that makes Conductor powerful

What is a Virtual Model Runner?

A Virtual Model Runner (VMR) is a unified endpoint that abstracts one or more physical model runners behind a single API. Applications connect to the VMR without knowing which backend will serve their request.

How It Works

  • Endpoint Aggregation: Combine multiple Ollama, vLLM, or OpenAI endpoints into one virtual endpoint
  • Transparent Routing: Conductor routes requests to healthy backends based on your load balancing strategy
  • Configuration Application: Pinned configurations are automatically merged into requests
  • Model Validation: Strict mode ensures only approved models can be accessed

Configuration Options

  • API Type: Expose as OpenAI or Ollama compatible API
  • Load Balancing: Round-robin, random, or first-available
  • Capabilities: Enable/disable embeddings, completions, model management
  • Strict Mode: Restrict to explicitly defined models only
  • Request Timeout: Configurable timeout for backend requests

Reliability & Performance

Built for high availability and optimal resource utilization

Load Balancing Strategies

  • Round Robin: Distribute requests evenly across healthy endpoints with optional weighting
  • Random: Weighted random distribution for unpredictable load patterns
  • First Available: Always route to the first healthy endpoint for consistent latency

Continuous Health Monitoring

  • Configurable health check intervals (default: 5 seconds)
  • Custom health check URLs and HTTP methods
  • Healthy/unhealthy thresholds for stability
  • Real-time health status via API and dashboard

Capacity Management

  • Max parallel requests per endpoint (rate limiting)
  • In-flight request tracking
  • Automatic 429 responses when at capacity
  • 502 Bad Gateway when no healthy endpoints

Get Started

Up and running in minutes with Docker or .NET

Docker (Recommended)

# Pull and run with Docker
docker run -d \
  --name conductor \
  -p 9000:9000 \
  -v conductor-data:/app/data \
  jchristn77/conductor:latest
# Or use Docker Compose
git clone https://github.com/jchristn77/conductor.git
cd conductor/docker
docker-compose up -d

.NET

# Clone and build
git clone https://github.com/jchristn77/conductor.git
cd conductor/src/Conductor.Server
dotnet build
dotnet run

Requires .NET 8.0 SDK
